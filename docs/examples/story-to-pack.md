# story_to_pack：小说/剧本到 AI 漫剧前期制作包

## 场景定位

`story_to_pack` 是 AI-native 垂直能力服务平台的第一个落地例子。

它不是平台本身，也不是项目的全部边界。它只是用来验证平台方法的一条具体服务单元：

```text
小说/剧本
-> 标准输入
-> 后端生产服务
-> 标准资源包
-> 多渠道调用
```

## 目标

把小说/剧本变成可评审、可分工、可继续生成的视频前期制作包。

它不承诺直接生成发布级视频。它交付的是“可开工”的生产资料。

## 输入

```text
- 小说/剧本文本
- 画风
- 目标镜头数
- 输出类型
```

可选输入：

```text
- 角色参考图
- 场景参考图
- 禁用风格
- 目标时长
- 旁白风格
```

## 输出

```text
package.zip
├── storyboard.md
├── characters.json
├── scenes.json
├── shots.csv
├── shots.json
├── prompts.json
├── assets/
│   ├── characters/
│   ├── scenes/
│   └── keyframes/
├── rough-cut.mp4
└── manifest.json
```

## 资源包价值

```text
小说/剧本
-> 可评审的方向
-> 可报价的镜头清单
-> 可分工的生产任务
-> 可复用的角色/场景资产
-> 可继续图生视频的关键帧
-> 可用于提案的粗剪 Demo
```

用户拿到资源包后，可以用于：

1. 内部立项评审。
2. 给客户或 IP 方确认方向。
3. 给制作团队拆任务。
4. 给 AI 操作员批量生成素材。
5. 给剪辑师做粗剪。
6. 估算成本、周期和返工风险。

## 软件项目类比

| 软件项目 | 视频前期制作包 |
|---|---|
| PRD | 故事分析、剧本结构、角色关系、场景设定 |
| 架构设计 | 分集、分场、分镜结构 |
| 技术方案 | 画风、模型选择、prompt、参考图、生成参数 |
| UI 设计稿 | 角色基准图、场景图、镜头关键帧 |
| 任务分解 | 每个镜头的生成任务、剪辑任务、配音任务 |
| 原型 Demo | 粗剪视频 |
| 工期/预算评估 | 镜头数量、生成成本、返工次数、资产复用情况 |

准确卖点不是：

> 一键生成视频。

而是：

> 把小说/剧本转成可开工的视频前期制作包。

## 生产流程

```text
上传小说/剧本
-> 文本拆解
-> 角色卡
-> 场景卡
-> 分镜表
-> 图片 prompt
-> 角色/场景/关键帧
-> 粗剪
-> package.zip
```

## 推荐调用方式

CLI：

```bash
ainong run story_to_pack ./story.txt --style anime --max-shots 30
ainong status <job_id>
ainong export <project_id> --format zip
ainong check <package.zip>
```

MCP 工具：

```text
create_story_pack
get_job_status
export_package
check_package
```

## 验收标准

功能验收：

1. 能从文本生成角色卡。
2. 能从文本生成场景卡。
3. 能生成分镜表。
4. 能生成图片 prompt。
5. 能生成至少一种图片资产。
6. 能导出标准 `package.zip`。
7. 能生成 `manifest.json`。

商业验收：

1. 至少 5 个真实文本完成试跑。
2. 至少 2 个用户愿意继续合作或付费。
3. 用户能明确说出资源包节省了哪部分工作。
4. 用户愿意把资源包用于真实项目评审或生产。

## 30 天样板目标

第一周先做 2-3 个样板资源包：

```text
样本 1：都市情感短剧
样本 2：玄幻升级流网文
样本 3：甜宠/复仇类短剧
```

每个样板都要包含：

```text
原文片段
分镜表
角色卡
角色基准图
场景图
镜头关键帧
粗剪 MP4
package.zip
```
